ES6 的方法来判断 0.1+0.2 == 0.3

dyygtfx Updated at Aug 15, 2016

0.1+0.2==0.3

在数学上来说是正确的，但是在 JS 里面这个确是不等的。这是由于JS 的浮点数精度问题，所以一直以来人们也是用了很多办法来判断0.1+0.2==0.3 的问题。

但是现在 ES6 Number 的一个新特性 EPSILON 解决了这个问题。Number.EPSILON 是一个静态属，直接用就可以。引入一个这么小的常量的目的，在于为浮点数计算，设置一个误差范围，如果计算两个数的误差能够小于Number.EPSILON，我们就可以认为得到了正确结果。

所以现在可以这么简单的判断 0.1+0.2 == 0.3 了：

function numbersCloseEnoughToEqual(n1,n2) {
  return Math.abs( n1 - n2 ) < Number.EPSILON;
}

numbersCloseEnoughToEqual(0.1+0.2,0.3)
=> true
